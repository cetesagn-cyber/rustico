🎯 QUÉ SUCEDE CUANDO EJECUTAS
───────────────────────────────────────────────────────────────────────────────

1. Docker descarga imágenes base (Node.js, Nginx)
2. Se construyen 2 contenedores (backend + frontend)
3. Se crea volumen SQLite
4. Se ejecutan migraciones automáticas
5. Se genera seed con datos de prueba
6. Backend escucha en :3001
7. Frontend (Nginx) escucha en :80
8. Todo conectado en red privada de Docker

Tiempo total: 2-5 minutos (primera vez)
Siguientes: 10-30 segundos

🔧 COMANDOS ÚTILES

Con script helper (Windows):
   rustico.bat start              # Inicia la app
   rustico.bat stop               # Detiene la app
   rustico.bat logs               # Ver logs
   rustico.bat restart            # Reinicia
   rustico.bat clean              # Limpia todo
   rustico.bat health             # Verifica salud

   ./rustico.sh start
   ./rustico.sh stop
   ./rustico.sh logs
   ./rustico.sh restart
   ./rustico.sh clean
   ./rustico.sh health

Manual:
   docker-compose ps              # Ver estado
   docker-compose logs -f         # Ver logs
   docker-compose restart         # Reiniciar
   docker-compose down            # Parar
   docker-compose down -v         # Parar y borrar BD

❓ PREGUNTAS FRECUENTES
───────────────────────────────────────────────────────────────────────────────

P: ¿Funciona en Windows?
R: Sí, con Docker Desktop. Usa rustico.bat

P: ¿Funciona en Mac/Linux?
R: Sí, con Docker. Usa rustico.sh

P: ¿Cuánto espacio ocupa?
R: ~500MB para imágenes + BD (<50MB)

P: ¿Se pierden datos si apago Docker?

P: ¿Puedo cambiar puertos?
R: Sí, edita docker-compose.yml

P: ¿Cómo accedo a la BD?
R: docker-compose exec backend sqlite3 /app/data/rustico.db

P: ¿Qué hago si falla?
